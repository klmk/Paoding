# AI 实施规格书（AIS）模板

> **定位：** AIS（AI Implementation Specification）是 PRD 到代码之间的翻译层。PRD 是给人看的（自然语言、允许模糊），AIS 是给 AI 看的（结构化、不允许模糊）。
> **生成方式：** AI 基于 PRD 自动生成，产品经理审核确认。
> **使用方式：** Phase 2/3 中 AI 基于 AIS（而非 PRD）进行架构设计和编码实现。

---

## 0. 需求来源信息

> **本章节记录需求的输入来源，帮助 AI 理解需求的上下文和置信度。**

| 项目 | 内容 |
|------|------|
| **来源类型** | 书面化文字 / 口述（语音转文字） / 图片/截图 / 视频 / 混合来源 |
| **原始材料** | [链接或引用原始需求材料] |
| **置信度** | 高（需求明确无歧义）/ 中（基本明确，部分待确认）/ 低（多处需要确认） |
| **预处理说明** | [描述对原始需求做了哪些预处理，如口述语义还原、图片 OCR 等] |
| **待确认项** | 1. [待确认的问题] 2. [待确认的问题] ... |
| **强调点** | [从语气分析、重复表述中识别出的高优先级需求点] |
| **被放弃的需求** | [口述中提到但后来放弃的需求，记录以备后续参考] |

**使用说明：**
- 如果需求来源为书面化文字且清晰明确，本章节可以简化为"来源类型：书面化文字，置信度：高"
- 如果需求来源为口述，本章节必须完整填写，特别是"预处理说明"和"待确认项"
- 如果需求来源为图片/视频，需说明视觉解析的结果和可能的信息损失
- 待确认项必须在 Phase 1 需求对齐阶段逐一确认

## 使用说明

1. **生成时机：** Phase 1（PRD 确认）完成后，进入 Phase 2 之前
2. **生成方式：** AI 基于 PRD 自动生成 AIS，产品经理逐项审核
3. **审核重点：** 检查 AI 对 PRD 的理解是否正确——AIS 中写的就是 AI 的理解
4. **确认后：** AIS 作为 Phase 2/3 的直接输入，替代 PRD 作为开发的精确规格
5. **变更管理：** PRD 变更时必须同步更新 AIS，AIS 变更时必须重新确认
