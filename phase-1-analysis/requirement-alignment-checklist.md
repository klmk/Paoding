# 需求对齐确认清单

> **定位**：本文件是 Phase 1 需求对齐确认（步骤 6）的检查工具。AI 在向使用者提交 PRD 文档进行确认时，必须逐项通过本清单的检查。
> **适用范围**：Phase 1 步骤 6（需求对齐确认）和 Gate 1 门控检查。
> **使用方式**：AI 逐项自检，每项必须标记为"通过"、"不通过"或"不适用"。"不通过"项必须处理后重新检查。

---

## 使用说明

### 检查流程

```
AI 自检（逐项检查） → 标记结果 → 处理不通过项 → 提交使用者确认 → 使用者反馈 → 更新 PRD → 重新检查 → 全部通过
```

### 检查结果标记

| 标记 | 含义 | 处理方式 |
|------|------|---------|
| **通过** | 该项检查已满足 | 无需处理 |
| **不通过** | 该项检查未满足 | 必须处理后重新检查 |
| **不适用** | 该项检查与当前项目无关 | 记录不适用原因 |

### 角色分工

- **AI**：负责逐项自检，确保所有检查项在提交给使用者之前已满足
- **使用者**：负责最终确认，确认 PRD 内容与自身需求一致
