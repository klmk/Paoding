# 项目回顾报告（Retrospective Report）

> **定位：** 本模板参考 BMad 的回顾机制，用于在 Phase 4 知识沉淀阶段对项目进行系统性回顾。回顾覆盖 Phase 0 至 Phase 3 的全部阶段，从全局视角分析成功经验、问题模式、技术债务和AI能力成长。
>
> **核心理念：** 回顾不是追责，而是学习。回顾的目标不是找出"谁做错了"，而是找出"什么可以做得更好"。每一次回顾都是下一次改进的起点。
>
> **填写指引：** 本模板由 AI 在 Phase 4 阶段填写。每个章节必须基于客观证据（文档记录、测试报告、审查记录等），不得使用模糊表述。
